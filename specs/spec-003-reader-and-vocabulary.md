# Spec 003：书架、阅读器与生词本

## 1. 目标
定义第一版核心阅读体验，覆盖：
- 书架即首页
- 绘本进入与继续阅读
- 阅读器翻页
- 点单词发音
- 点整句播放
- 单词释义弹层
- 生词本与阅读进度

该 spec 直接决定 MVP 是否“可用”。

---

## 2. 已确认约束
- 书架即首页，不单独做首页
- 书架展示方式：双列网格
- 点击绘本卡片后直接进入阅读页
- 阅读时英文正文固定放在图片下方
- 翻页方式：同时支持左右滑动和点击左右区域
- 生词本入口：阅读页右上角
- 阅读到最后一页即算读完
- 生词本第一版要做
- 生词本全局去重，同一个词只保留一条
- 生词本第一版不显示来源书名
- 生词删除需要二次确认
- 第一版不做跟读
- 第一版不做家长端

---

## 3. 范围
### 3.1 本 spec 包含
- 书架首页
- 最近阅读
- 阅读器页面
- 左右翻页
- 单词点击与释义弹层
- 整句音频播放
- 生词收藏 / 查看 / 删除
- 阅读进度保存与恢复

### 3.2 本 spec 不包含
- 书籍搜索
- 推荐算法
- 跟读与口语评测
- 整页中文翻译
- 家长报告
- 学习计划
- 复习算法
- 多设备同步

---

## 4. 页面设计

## 4.1 书架首页
### 页面目标
用户登录后直接看到可读绘本，并快速继续上次阅读。

### 页面模块
1. 顶部栏
   - 页面标题
   - 可预留个人入口

2. 最近阅读模块
   - 有记录时展示最近阅读卡片
   - 点击后进入该书上次阅读页

3. 绘本网格区
   - 双列网格
   - 展示封面、标题、分级
   - 可展示简化进度

### 空状态
- 没有最近阅读记录时，不显示最近阅读模块
- 仍然展示全部绘本

---

## 4.2 阅读器页面
### 页面布局
1. 顶部栏
   - 返回按钮
   - 书名 / 页码进度
   - 生词本入口

2. 内容图片区
   - 当前页图片

3. 正文区
   - 英文正文
   - 支持逐词点击
   - 支持整句播放按钮

4. 单词释义弹层
   - 单词文本
   - 中文释义
   - 可选音标
   - 播放发音按钮
   - 加入生词本按钮

---

## 4.3 生词本页面
### 展示内容
- 单词
- 中文释义
- 发音按钮
- 删除按钮

### 明确不显示
- 来源书名
- 来源页码

---

## 5. 用户流程

### 5.1 打开一本书
1. 用户进入书架首页
2. 点击绘本卡片
3. 系统检查是否有阅读记录
4. 有记录则跳转到上次阅读页
5. 无记录则从第一页打开

### 5.2 阅读与翻页
1. 用户在阅读页查看图片与英文正文
2. 左右滑动可翻页
3. 点击左右区域也可翻页
4. 每次翻页后保存当前页进度
5. 到最后一页时标记为已读完成

### 5.3 点单词
1. 用户点击正文中的某个单词
2. 打开释义弹层
3. 可查看释义、播放单词发音
4. 可加入生词本

### 5.4 点整句
1. 用户点击当前页整句播放按钮
2. 播放当前页句子音频

### 5.5 查看生词本
1. 用户点击阅读页右上角生词本入口
2. 进入生词本页
3. 查看已收藏词条
4. 可播放发音
5. 删除时弹出二次确认

## 6. 数据模型

### 6.1 书架项
```kotlin
data class BookShelfItem(
    val bookId: String,
    val title: String,
    val coverAsset: String,
    val level: String,
    val pageCount: Int,
    val readProgress: Int,
    val lastReadPage: Int?,
    val isFinished: Boolean,
)
```

### 6.2 阅读进度
```kotlin
data class ReadingProgress(
    val userId: String,
    val bookId: String,
    val currentPage: Int,
    val finished: Boolean,
    val updatedAt: Long,
)
```

### 6.3 生词项
```kotlin
data class VocabularyItem(
    val userId: String,
    val wordId: String,
    val normalizedWord: String,
    val word: String,
    val meaningZh: String,
    val phonetic: String? = null,
    val audioAsset: String? = null,
    val createdAt: Long,
)
```

说明：
- 生词本全局去重建议以 `userId + normalizedWord` 作为唯一键
- 第一版不要求保留多个来源书页

## 7. 交互规则

### 7.1 阅读进度规则
- 打开某书时优先恢复上次阅读页
- 翻页后立即更新当前页码
- 到最后一页即视为已读完成
- 已读完成的书仍可再次进入阅读

### 7.2 生词本规则
- 同一个词重复收藏时，不新增新记录
- 已收藏时应给出“已加入生词本”提示
- 删除生词前必须二次确认

### 7.3 点词规则
- 点击目标词后高亮
- 弹层不应遮挡主要操作区域过多
- 若单词没有音频资源，应允许只显示释义

---

## 8. 存储建议
### 8.1 本地存储
第一版建议使用 `Room` 保存：
- `ReadingProgress`
- `VocabularyItem`

### 8.2 设计原因
- 内测版先保证稳定可用
- 账号虽已存在，但第一版不强依赖远程同步
- 后续需要云同步时，可在 repository 层增加 remote data source

---

## 9. 验收标准
1. 登录后进入书架首页，而不是额外首页。
2. 书架采用双列网格展示绘本。
3. 点击绘本卡片后能直接进入阅读页。
4. 阅读页能同时支持滑动翻页和点击区域翻页。
5. 英文正文固定显示在图片下方。
6. 点击单词后能弹出释义层，并支持发音与收藏。
7. 点击整句播放按钮后能播放当前页句子音频。
8. 阅读进度能正确恢复。
9. 到最后一页后该书被标记为已读完成。
10. 生词本中同一单词不会重复出现。
11. 删除生词时必须二次确认。

---

## 10. 风险与后续扩展
### 10.1 风险
- 页面文本与单词索引不对齐会直接影响点词体验。
- 图片区、正文区、弹层区如果布局过挤，会影响儿童使用。
- 若 TTS 单词音频与句子音频风格差异过大，体验会割裂。

### 10.2 后续扩展方向
- 底部 Tab 或独立“我的”页
- 生词本分类与搜索
- 云端阅读记录同步
- 跟读、口语评测
- 阅读数据分析

---

## 11. 当前未决问题
以下问题还需要在进入 UI 细化或实现前再确认：
- 书架卡片是否显示明确百分比进度，还是只显示“已读 / 未读 / 继续阅读”
- 阅读器正文区是否支持字体大小调整
- 生词本页面是否需要批量删除
- 书架首页是否需要个人中心入口以及入口位置
